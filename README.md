# 🧪 SauceDemo Manual Testing Project

## 📌 Project Overview

This project demonstrates end-to-end manual testing performed on the SauceDemo e-commerce web application. The main objective of this testing activity is to validate critical user workflows such as login, product selection, cart management, and checkout functionality.

---

## 🎯 Project Objective

* Verify the Login functionality with valid and invalid credentials
* Validate Add to Cart operations
* Test complete Checkout workflow
* Ensure payment process functionality
* Identify functional defects and usability issues

---

## 🔎 Scope of Testing

* Login Testing
* Product Selection Testing
* Cart Management Testing
* Checkout Process Testing
* Payment Validation
* Negative Testing

---

## 🧾 Test Deliverables

* Test Summary Report
* Detailed Test Cases
* Test Execution Results
* Bug Report with Severity Classification

---

## 🐞 Defect Identified

**Severity:** Critical Blocker

During checkout testing, a payment gateway issue was identified:

* Error message displayed: *"Credit and debit card payments aren’t available right now"*
* Pay Now button remained stuck in **Processing...** state
* User was unable to complete the purchase transaction

This issue directly impacts business revenue and prevents successful order completion.
